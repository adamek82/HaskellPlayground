# HaskellPlayground

A small playground for learning and experimenting with Haskell.

At the moment, this project is intentionally minimal. It starts with a simple `Hello, Haskell!` executable and will be gradually extended with small functions, language experiments, and notes while learning Haskell.

## Environment

The project was created and tested on:

* Ubuntu 24.04 running under WSL
* GHC installed via GHCup
* Cabal as the build tool
* Visual Studio Code with the official Haskell extension

The WSL part is not essential, but the setup below assumes a Linux/Ubuntu environment.

## Installing the Haskell toolchain

Install basic system dependencies first:

```bash
sudo apt update
sudo apt install -y build-essential curl libffi-dev libgmp-dev libncurses-dev pkg-config
```

Install GHCup:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://get-ghcup.haskell.org | sh
```

During the interactive GHCup setup, use the following choices:

```text
Base channel: GHCup maintained
Pre-releases channel: No
Cross channel: No
3rd party channel: No
Install Haskell Language Server: Yes
Enable better Stack integration with GHCup: Yes
```

Notes:

* Use the default stable GHCup-maintained channel.
* Do not enable pre-release/cross/third-party channels for a simple learning setup.
* Install Haskell Language Server, especially if you plan to use VS Code.
* Stack is optional, but if it is installed, letting it use GHCup-managed GHC versions keeps the setup cleaner.

After installation, either restart the terminal or load the GHCup environment manually:

```bash
. ~/.ghcup/env
```

Verify the installation:

```bash
ghcup --version
ghc --version
ghci --version
cabal --version
stack --version
haskell-language-server-wrapper --version
```

You can also test GHCi:

```bash
ghci
```

Inside GHCi:

```haskell
2 + 2
:t map
:t foldr
:quit
```

## Creating the Cabal project

The project was initialized with:

```bash
cabal init --interactive
```

The relevant choices were:

```text
Package type: Executable
Cabal specification: 3.0
Package name: haskell-playground
Package version: 0.1.0.0
License: MIT
Author: adamek82
Homepage: empty
Category: empty
Main module: Main.hs
Language: GHC2021
Add informative comments to the .cabal file: No
```

The generated structure is:

```text
HaskellPlayground/
├── app/
│   └── Main.hs
├── CHANGELOG.md
├── LICENSE
├── haskell-playground.cabal
└── README.md
```

## Building and running

Run the project with:

```bash
cabal run
```

Expected output:

```text
Hello, Haskell!
```

You can also open an individual file in GHCi or run a standalone file with `runghc`, but the main project workflow uses Cabal.

## VS Code setup

Install the official VS Code extension:

```text
Haskell
```

Extension identifier:

```text
haskell.haskell
```

When VS Code asks how to manage or discover HLS and the relevant toolchain, choose:

```text
Automatically via GHCup
```

When working in WSL, make sure VS Code is opened in the remote WSL context. The lower-left corner of VS Code should show something like:

```text
WSL: Ubuntu-24.04
```

The Haskell extension should be installed inside the WSL remote environment, not only locally on Windows.

## VS Code settings

The following settings were needed to make HLS/toolchain discovery and semantic syntax highlighting work properly:

```json
{
    "haskell.manageHLS": "GHCup",
    "editor.semanticHighlighting.enabled": true,
    "haskell.plugin.semanticTokens.globalOn": true
}
```

Depending on the setup, these may need to be added either to the normal user settings or to the remote WSL settings.

Useful commands in VS Code:

```text
Preferences: Open User Settings (JSON)
Preferences: Open Remote Settings (JSON)
Developer: Reload Window
```

A good quick check that HLS works is to hover over a function such as `putStrLn`. VS Code should show type information similar to:

```haskell
putStrLn :: String -> IO ()
```

## Git ignore

The project should not commit Cabal build output. In particular, `dist-newstyle/` is generated by Cabal and should stay ignored.

A minimal `.gitignore` for this project:

```gitignore
# Cabal build output
dist-newstyle/
dist/

# GHC build artifacts
*.hi
*.o
*.dyn_hi
*.dyn_o

# Profiling / coverage artifacts
*.prof
*.hp
*.eventlog
*.tix
.hpc/

# Cabal local/user-specific config
cabal.project.local
cabal.project.local~
.ghc.environment.*

# Stack, if used later
.stack-work/

# Editor / OS noise
.vscode/
.idea/
.DS_Store
Thumbs.db
```
